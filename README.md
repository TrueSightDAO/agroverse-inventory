# agroverse-inventory

Public **store inventory snapshot** for [agroverse.shop](https://agroverse.shop): SKU product IDs → numeric stock counts. Data is **generated**—do not hand-edit the JSON.

## Canonical files

| | |
|---|---|
| **Branch** | `main` |
| **Store snapshot** | [`store-inventory.json`](store-inventory.json) |
| **Store URL (browsers / `fetch`)** | `https://raw.githubusercontent.com/TrueSightDAO/agroverse-inventory/main/store-inventory.json` |
| **Partner snapshot** | [`partners-inventory.json`](partners-inventory.json) |
| **Partner URL** | `https://raw.githubusercontent.com/TrueSightDAO/agroverse-inventory/main/partners-inventory.json` |
| **Currencies list** | [`currencies.json`](currencies.json) |
| **Currencies URL** | `https://raw.githubusercontent.com/TrueSightDAO/agroverse-inventory/main/currencies.json` |

`partners-inventory.json` is keyed by partner page slug (`partner_id`). Each item includes ledger counts at the partner plus `onlineInventory` / `availableOnline` derived from the same store snapshot as `store-inventory.json`. It is generated by `market_research/scripts/sync_agroverse_store_inventory.py` (same run as the store file). **There is no separate GitHub Action in this repo** today; publish by running that script with credentials and pushing `main`, or extend your existing automation (GAS / Edgar) if you want continuous updates.

## JSON shape

```json
{
  "generatedAt": "2026-04-07T19:56:10.893Z",
  "source": "update_store_inventory",
  "inventory": {
    "oscar-bahia-ceremonial-cacao-200g": 68,
    "...": 0
  }
}
```

Consumers should read **`inventory`** (map of product id → count). `generatedAt` is ISO-8601 UTC from the last successful publish.

**`currencies.json`** is an alphabetized list of **Main Ledger** currency strings (column **A** of the **Currencies** sheet in the main Agroverse spreadsheet). It is updated by the **`repackaging-currency-ingest`** Apps Script (`gas/repackaging-currency-ingest/`) when an operator submits from the repackaging planner ([`dapp/repackaging_planner.html`](https://dapp.truesight.me/repackaging_planner.html)). Script properties: **`AGROVERSE_INVENTORY_PUBLISH_SECRET`**, **`AGROVERSE_INVENTORY_CURRENCIES_PAT`** (or **`AGROVERSE_INVENTORY_GIT_REPO_UPDATE_PAT`**). Run **`authorizeUrlFetchForRepackagingIngest()`** once after deploy to grant `UrlFetchApp` + spreadsheet access.

Each successful ingest also writes **`currency-compositions/{request_id}.json`** with the full batch: `inputs[]` (qty, unit cost, extended), `outputs[]` (label, units M, weight-per-unit g, allocated unit cost USD, line total USD), `totals` (inputs subtotal, grand total, total output weight g, cost per gram USD), `holder`, and optional `proof_of_work`. Override the folder with Script property **`REPACKAGING_COMPOSITIONS_DIR`**. Example URL: `https://raw.githubusercontent.com/TrueSightDAO/agroverse-inventory/main/currency-compositions/<request_id>.json`.

One planner submit = one POST, and the ingest writes:

- **`Telegram Chat Logs`** (ops workbook): **one row** per request — the whole sanitized batch JSON.
- **`Currency Creation`** (ops workbook): **N rows** per request — one per output, keyed `{request_id}#{index}` in column B. Dedup recognizes both exact `{request_id}` (legacy single-output) and prefix `{request_id}#`.
- **[`Currencies`](https://docs.google.com/spreadsheets/d/1GE7PUq-UT6x2rBN-Q2ksogbWpgyuh2SaxJyG_uEK6PU/edit?gid=1552160318#gid=1552160318)** (Main Ledger): **N rows** per request, sorted by column A after insert:
  - **Column A**: `suggested_currency` (with `| YYYYMMDD` auto-appended by the DApp if no date is present, to avoid collisions).
  - **Column B** (default, **`CURRENCIES_UNIT_COST_USD_COLUMN`**): allocated **`unit_cost_usd`** for that output — *grand inputs cost ÷ total output weight in grams × this output's per-unit weight in grams*. Header: **Unit cost USD**. Set the property to `0` to skip.
  - **Column N** (default, **`CURRENCIES_RAW_REQUEST_COLUMN`**): `raw_request_text` — a plain-text dump of the whole submit (holder, every input with qty/unit/ext, every output with label/M/weight/unit-cost/line-total, totals). Header: **Raw request text**. Set the property to `0` to skip.
  - **Column O** (default, **`CURRENCIES_COMPOSITION_JSON_URL_COLUMN`**): raw GitHub URL for `currency-compositions/{request_id}.json`. Header: **Composition JSON**. Set the property to `0` to skip. If this property resolves to the same column as `CURRENCIES_RAW_REQUEST_COLUMN`, the URL write is skipped (raw text wins).

**Default change:** the composition-URL column default was **14 (col N)** in schema v1. It is now **15 (col O)** so col N can hold the raw request text. If your deployment was relying on the old default, either set `CURRENCIES_COMPOSITION_JSON_URL_COLUMN` explicitly to the column you want (any free column ≠ N), or accept the new col O default.

## How it is updated

1. **Google Apps Script** project **`update_store_inventory`** ([editor](https://script.google.com/home/projects/1P0Mg33i_dD9x9IeoHYvtKrf0xFcmUznpqAswyC_KXR3VJZu-0C-UOP0v/edit)) recomputes counts from DAO ledgers and **Agroverse SKUs** (see tokenomics clasp mirror under `tokenomics/clasp_mirrors/1P0Mg33i…/`).
2. It pushes this file via the **GitHub Contents API** using a fine-grained PAT stored only in **Apps Script → Project Settings → Script properties** as **`AGROVERSE_INVENTORY_GIT_REPO_UPDATE_PAT`**. **Never commit that token to this repo.**
3. OAuth: **`appsscript.json`** must include **`https://www.googleapis.com/auth/script.external_request`**; the script owner must authorize external requests (see `authorizeUrlFetchForSnapshot()` in the GAS project).
4. Triggers: **time-driven** `updateStoreInventory` (e.g. hourly) and **HTTP** actions `publishInventorySnapshot` / `recalculateAndPublishInventory` (shared secret **`AGROVERSE_INVENTORY_PUBLISH_SECRET`**).
5. **sentiment_importer** (Edgar) may enqueue **`AgroverseInventorySnapshotPublishWorker`** after Meta/QR checkout and after successful ledger **`WebhookTriggerWorker`** runs (`dao_controller#trigger_immediate_processing`). Configure **`AGROVERSE_INVENTORY_GAS_WEBAPP_URL`** and **`AGROVERSE_INVENTORY_PUBLISH_SECRET`** on the Edgar host.

Commits may be **skipped** when the `inventory` map is unchanged (no useless churn).

## Shop frontend

**agroverse_shop** loads inventory via **`js/inventory-service.js`**: prefers this **raw GitHub** URL, falls back to the GAS **`getInventory`** endpoint if the fetch fails.
